# Naming Rules

Naming conventions and rules for the project.

## Folder Naming <!-- id: naming-rule.folder -->

> Rules for naming folders across the project.

- Use lowercase `kebab-case` for all folder names (e.g., `user-auth`, `api-routes`).
- Avoid abbreviations unless they are widely understood (e.g., `api`, `db`).
- Group related files under a descriptive folder name that reflects the module or feature.
